# Quality Score — PaperJetSim

각 모듈이 "완성"되었다고 볼 수 있는 기준을 정의한다.
코딩 후 이 체크리스트를 통과해야 다음 모듈로 넘어간다.

---

## 훅 파이프라인 (자동 실행)

코드 수정 후 `git commit` 시 아래 3단계가 자동으로 실행된다.
**모두 통과해야 커밋이 허용된다.** 상세 설정 → `docs/references/dev-hooks.md`

```
[1] ruff --fix       코드 정리    미사용 import·변수 제거, 포맷 통일
[2] vulture          죽은 코드    호출 안 되는 함수·클래스 탐지
[3] pytest           테스트       단위 테스트 전체 실행
```

수동 실행:
```bash
pre-commit run --all-files   # 전체 훅 한 번에
ruff check . --fix           # 코드 정리만
vulture . whitelist.py       # 죽은 코드만
pytest -q                    # 테스트만
```

## Definition of Done (완료 기준 원칙)

모든 모듈은 아래 5가지 조건을 모두 충족해야 완료로 간주한다.

1. **훅 파이프라인 통과** — ruff + vulture + pytest 모두 GREEN (커밋 시 자동 확인)
2. **단위 테스트 통과** — test-cases.md 기준 해당 모듈 TC 전체 GREEN
3. **수동 동작 확인** — 아래 각 모듈별 체크리스트 항목 직접 확인
4. **60fps 유지** — 해당 모듈 활성화 시 프레임드롭 없음
5. **예외 처리** — 비정상 입력(극단값, 0, 음수 등)에서 크래시 없음

## 통합 테스트 체크리스트

전체 게임 루프가 완성된 후 수행하는 end-to-end 확인.

- [ ] **정상 플레이 루프**: 앱 실행 → 발사 준비 → 발사 → 착지 → 결과 → 재시작 5회 반복, 크래시 없음
- [ ] **극단값 연속 발사**: 힘 100% + 각도 45° + 순풍 최대 → 10회 발사, 정상 동작
- [ ] **최소값 연속 발사**: 힘 0% + 각도 -10° + 역풍 최대 → 5회 발사, 즉시 착지 정상 처리
- [ ] **장시간 실행**: 30분 연속 실행, 메모리 누수(RAM 증가) 없음
- [ ] **빠른 재시작**: 비행 중 R키 → 즉시 초기화 → 재발사, 상태 꼬임 없음
